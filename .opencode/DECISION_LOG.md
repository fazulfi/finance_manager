# DECISION_LOG.md — Architectural Decisions

> All significant technical decisions are logged here. Updated by orchestrator after each decision.

| Date       | Decision                                                                                                           | Reason                                                                                                                                                                               | Affected Files                                                      |
| ---------- | ------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------- |
| 2026-04-11 | Monorepo with Turborepo + pnpm workspaces                                                                          | Shared code between web and mobile, single pipeline                                                                                                                                  | `turbo.json`, `pnpm-workspace.yaml`                                 |
| 2026-04-11 | MongoDB + Prisma (not PostgreSQL + SQLAlchemy)                                                                     | Blueprint specifies MongoDB; Prisma gives type-safe queries                                                                                                                          | `packages/db/prisma/schema.prisma`                                  |
| 2026-04-11 | tRPC v10 for API layer                                                                                             | End-to-end type safety between Next.js and client, no separate REST schema                                                                                                           | `packages/api/`                                                     |
| 2026-04-11 | NextAuth.js v5 for auth                                                                                            | First-class Next.js App Router support, Prisma adapter available                                                                                                                     | `apps/web/app/api/auth/`                                            |
| 2026-04-11 | shadcn/ui as component base                                                                                        | Unstyled primitives, full ownership of component code, Tailwind-native                                                                                                               | `packages/ui/`                                                      |
| 2026-04-11 | Expo Router (file-based) for mobile                                                                                | Same mental model as Next.js App Router, reduces context switching                                                                                                                   | `apps/mobile/app/`                                                  |
| 2026-04-11 | Turborepo v2 `"tasks"` key (not `"pipeline"`)                                                                      | Turborepo v2 changed the pipeline key from `pipeline` to `tasks`; using `pipeline` triggers deprecation warning and breaks in future versions                                        | `turbo.json`                                                        |
| 2026-04-11 | `noUncheckedIndexedAccess: true` in base tsconfig                                                                  | Finance app handles arrays of transactions, accounts, goals — index access must always be null-checked to prevent runtime errors on missing data                                     | `packages/tsconfig/base.json`                                       |
| 2026-04-11 | `packages/ui` extends `nextjs.json` not `base.json`                                                                | UI package contains JSX/TSX components; `nextjs.json` config provides `jsx: preserve` and browser DOM lib types required for React components                                        | `packages/ui/tsconfig.json`                                         |
| 2026-04-11 | `react-native.json` uses `moduleResolution: Node` not `NodeNext`                                                   | Metro bundler (used by Expo/React Native) does not support NodeNext module resolution; using Node (legacy) resolution is required for Expo SDK 51 compatibility                      | `packages/tsconfig/react-native.json`                               |
| 2026-04-11 | `next-auth@5.0.0-beta.25` pinned to beta                                                                           | NextAuth v5 beta is required for Next.js 14 App Router compatibility; v4 does not support App Router server components and edge runtime                                              | `apps/web/package.json`                                             |
| 2026-04-11 | All intra-workspace deps use `workspace:*` protocol                                                                | pnpm workspace protocol ensures all packages resolve to local workspace copies during development and prevents version drift between workspace packages                              | All `package.json` files with `@finance/*` deps                     |
| 2026-04-11 | Downgraded @tanstack/react-query to ^4.36.1 (v4, not v5)                                                           | tRPC v10 peer dep requires @tanstack/react-query@^4.x; v5 has breaking API changes incompatible with tRPC v10 — must not upgrade tRPC to v11 per AGENTS.md                           | `apps/web/package.json`, `apps/mobile/package.json`                 |
| 2026-04-11 | Pinned react-native-reanimated to ~3.10.1 explicitly                                                               | NativeWind 4 resolved to reanimated 4.3.0 (requires RN 0.81-0.85) via transitive dep; Expo SDK 51 / RN 0.74.5 requires reanimated 3.x — explicit pin overrides transitive resolution | `apps/mobile/package.json`                                          |
| 2026-04-11 | MongoDB via Docker (not native install) for local dev                                                              | Docker provides isolated, reproducible MongoDB 7 environment on Windows; native MongoDB on Windows has PATH/service conflicts and is harder to reset                                 | `docker run finance-mongodb`                                        |
| 2026-04-11 | Write .env files using [System.IO.File]::WriteAllText with UTF8Encoding($false) on Windows                         | PowerShell Set-Content -Encoding UTF8 injects BOM (U+FEFF) which can corrupt dotenv key parsing; System.IO.File with UTF8Encoding(false) writes clean UTF-8                          | `packages/db/.env`                                                  |
| 2026-04-11 | Root `tsconfig.json` uses solution anchor pattern (`files:[]`, `references:[]`) without `composite:true`           | Leaf packages use `noEmit:true` which is incompatible with `composite:true`; Turborepo handles build orchestration instead of `tsc -b`                                               | `tsconfig.json`, `packages/*/tsconfig.json`, `apps/*/tsconfig.json` |
| 2026-04-11 | `prettier-plugin-tailwindcss` excluded from `prettier.config.js`                                                   | Package is not installed in any workspace; would cause a runtime error if referenced; can be added later as a separate step when Tailwind workspaces are wired up                    | `prettier.config.js`                                                |
| 2026-04-11 | ESLint v8 legacy format (`.eslintrc.js` with `module.exports`) used at root — not flat config (`eslint.config.js`) | `packages/eslint-config/index.js` exports ESLint v8 legacy format; mixing legacy and flat config formats would cause incompatibility                                                 | `.eslintrc.js`, `packages/eslint-config/index.js`                   |
| 2026-04-11 | Exclude entire `.opencode/` directory from git tracking (not just individual files)                                 | Keep AI agent workspace files (BLUEPRINT.md, AGENTS.md, DECISION_LOG.md, sessions, etc.) private/local — not publicly visible on GitHub; single directory rule is simpler and future-proof | `.gitignore` (line 70)                                              |
| 2026-04-11 | Orchestrator owns git push (Push Gate at Step 7); docs agent remains commit-only                                    | Prevent session commits from accumulating locally without reaching GitHub; single push at session end is safer than push-per-update; keeps push authority in one place (orchestrator) and preserves the docs agent commit-only constraint | `.opencode/AGENTS.md` (lines 94–138)                               |
